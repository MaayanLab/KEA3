Overview


About This Project
KEA2 (Kinase Enrichment Analysis 2) is a tool which takes an input of a gene list and outputs a list of associated kinases. This tool was originally built in 2009, but many kinase-substrate databases that were originally curated have expanded and new kinase-substrate databases have emerged. Thus, the purpose of the KEA3 project is to collect and process new data pertaining to kinase-substrate interactions and expand the KEA2 database, ultimately allowing for a more comprehensive understanding of the human kinome. Thus far, kinase-substrate interactions have been integrated from seven databases, and the data from each has been converted into GMT and SIG files, found in the "Kinase-Substrate Networks" tab.

What is a GMT file?
GMT files are used to describe gene sets, as each row corresponds to a different gene set. Each row lists a name (for example, a kinase in kinase-substrate interactions), a description, and the genes in the gene set (in this example, all of the substrates that interact with the kinase).

What is a SIG file?
SIG files represent gene interaction networks, also listed by row. In each row, the columns represent: source name, source human accession, source mouse accession, source type, source location, target name, target human accession, target mouse accession, target type, target location, effect, type of interaction, and lastly PubMed IDs. Some of these fields may be filled with 'NA' if information is not available to fill the column.

Other Files
For the kinase-substrate interaction networks, in addition to the GMT and SIG files (described above), a GMT file for the combined data will also be created. From this combined data file, another GMT file is included, in which genes are ranked according to their level of co-expression with the kinase. To be included in the GMT, genes must also meet a co-expression cut-off, so if only some genes meet this cut-off a kinase may have fewer entries than in the original GMT. The methods to determine this cut-off are included in corresponding R files.

What else is on this website?
In addition to kinase-substrate interactions, this website will also host data related to protein-protein interactions and transcription factor-target interactions, found respectively in the "Protein Interaction Networks (PPI)" and "Transcription Factor-Target Interactions". This data has been similarly compiled from different PPI and TF-target databases.

Where can I find additional information about these files?
For all three interaction types, the processed files and information regarding processing can be found in the corresponding tabs. Tables containing this information will also include statistics regarding the data in each GMT file. The Methods section below can provide a brief overview of the data processing.

Methods


Mining the Data
For the "Kinase-Substrate Networks", Python scripts were written for each kinase-substrate database to process the raw data in the GMT and SIG file formats. Jupyter Notebooks corresponding to these scripts can be found for each kinase-substrate database and include comments detailing the function of each code block for increased clarity. For the "Protein Interaction Networks (PPI)", there are also Jupyter Notebooks for each PPI database detailing the methods used to process the data.

Filtering the Data
Only kinases or proteins which had at least five substrates or interactions and no more than 2000 substrates or interactions are included in the GMT files. However, the SIG files include all kinase-substrate interactions or protein-protein interactions regardless of the total number for a given protein or kinase. Furthermore, data was also filtered by species, so the data reflects only kinases and proteins found in humans and mice.
